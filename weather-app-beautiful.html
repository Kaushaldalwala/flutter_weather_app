Complete Architecture & Component Documentation

1 main()
Initializes Flutter and ensures everything is ready before the app runs.
  • Loads .env file to get API keys for the weather service
  • If .env is missing, uses a default API key
  • Starts the app by running MyApp
2 MyApp (Stateless Widget)
Main container for the app that defines the overall structure and theme.
  • Sets the title, theme (dark + Material 3), and home page (WeatherPage)
  • Does not change during runtime
Stateless Material 3
3 WeatherPage (Stateful Widget)
A dynamic page where weather data updates based on user input or location.
  • Connects to _WeatherPageState where the actual logic and UI updates happen
  • Handles real-time weather data updates
Stateful Dynamic
4 _WeatherPageState
The core state management class that handles all app logic and data.
  • Holds all the state variables (city name, temperature, wind, etc.)
  • Manages user input (_cityController) and loading/error states
  • Contains methods for fetching weather, updating UI, and showing animations
State Management
5 initState()
Lifecycle method that runs once when the widget is first created.
  • Calls _fetchWeather() to get weather based on the user's current location
  • Initializes the app with location-based weather data
Lifecycle
6 _fetchWeather()
Handles location permission and fetches weather for user's current position.
  • Checks if the app has location permission
  • Requests permission if denied
  • Gets the user's current GPS location
  • Calls _fetchWeatherByCoordinates() to fetch weather for the location
  • Handles errors if location cannot be accessed
GPS Permissions
7 _fetchWeatherByCity(String cityName)
Fetches weather data from OpenWeather API using a city name.
  • Handles empty input validation
  • Manages "City not found" errors
  • Handles HTTP errors or connection failures
  • Updates the state variables with the fetched weather data
API Integration Error Handling
8 _fetchWeatherByCoordinates(lat, long)
Fetches weather data using GPS coordinates from the user's location.
  • Handles network or API errors gracefully
  • Updates the state variables if successful
Geolocation
9 _updateWeatherData(Map data)
Processes and formats the API response for display.
  • Extracts weather details from the API response
  • Converts numeric values to string format for display
  • Updates the UI dynamically using setState()
Data Processing
10 _getWeatherAnimation(String? condition)
Returns the path of a Lottie animation based on weather conditions.
  • Maps conditions: rain, snow, clear, clouds, etc.
  • Shows a default loader if no condition is available
Lottie Animations
11 _getWeatherIcon(String condition)
Returns a Material icon that matches the weather condition.
  • Example: clouds ‚Üí cloud icon
  • Example: rain ‚Üí water drop icon
Material Icons
12 _buildWeatherDetail(icon, label, value)
Creates a small card showing a single weather detail.
  • Displays Icon (Wind, Humidity, Feels Like)
  • Shows Label (text)
  • Shows Value (data from API)
  • Styled with semi-transparent background and rounded corners
UI Component
13 _buildLottieAnimation(String path)
Shows the Lottie animation for the current weather condition.
  • If animation fails to load, displays a fallback icon + weather text
  • Provides smooth animated weather visualizations
Animation Handler
14 build(BuildContext context)
Main layout of the app page that renders the complete UI.
  • AppBar with logo and title
  • Search bar to enter city names
  • Weather info section with city name, animation, temperature
  • Weather condition display
  • Details row (Wind, Humidity, Feels Like)
  • Handles loading spinner or error messages
  • Uses gradient background and padding to look neat
Main UI Build Method
